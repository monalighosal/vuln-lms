Insecure Deserialization Vulnerability

This page demonstrates insecure deserialization. Importing preferences from untrusted sources can lead to prototype pollution and code execution.

Import User Preferences
Enter JSON preferences to import
Example Malicious Payloads:
// Prototype pollution example:
{"theme": "dark", "__proto__": {"polluted": "vulnerable"}}
// More complex prototype pollution:
{"theme": "light", "constructor": {"prototype": {"polluted": "vulnerable"}}}
Security Explanation

This application is vulnerable to insecure deserialization because it uses JSON.parse() directly on user input without proper validation or sanitization.

How the vulnerability works:

  • User input is directly parsed using JSON.parse()
  • No validation or sanitization of the parsed object
  • Allows for prototype pollution attacks
  • Could potentially lead to code execution in some contexts

Impact: An attacker can manipulate object prototypes, potentially causing application instability, information disclosure, or remote code execution.
